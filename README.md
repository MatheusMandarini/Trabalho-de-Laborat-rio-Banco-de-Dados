# Trabalho-de-Laboratorio-Banco-de-Dados
Aplicar os conceitos e estudos da disciplina de Banco de Dados e Laboratório de Banco de
Dados para desenvolver um projeto para um banco de dados, com foco na implementação do banco
dedos, consultas e regras negócios para o tema definido a partir de uma modelagem conceitual
inicial definida na seção 2.
Os itens a serem desenvolvidos na atividade são:
1) Escrever o Cenário do Tema Escolhido
2) Projeto do Diagrama Entidade-Relacionamento para Banco de Dados
3) Projeto do Esquema Relacional para Banco de Dados
4) Script para criar Banco de Dados, tabelas e índices.
5) Script para povoar as tabelas: devem ser gerados no mínimo 30 registros por tabela.
6) Elaborar e implementar as consultas relevantes para o tema:
Número mínimo de consultas a serem feitas: 10 consultas. Ao todo as consultas devem
exemplificar os exemplos de: junção interna, restrição de tempo ou data, restrição de
string, operações entre conjuntos, funções agregadas, agrupamentos, restrição de
agrupamento, subconsultas, exemplificar o uso de operador/modificador IN, NOT,
EXIST, DISTINCT, entre outros.
7) Elaborar Gatilhos
Fazer no mínimo 3 gatilhos. Deve-se apresentar gatilhos para exemplificar geração de
histórico, automatização de regra de negócio, validação de dados, cálculos de atributos
derivados quando for o caso, entre outros exemplos.
8) Elaborar Procedimentos Armazenados:
Fazer no mínimo 3 procedimentos armazenados. Deve-se apresentar um procedimento
para a implementação de regras de negócios específica do tema, transição ou
encerramento de ciclo, realização de operação. Esses procedimentos não estão
relacionados aos gatilhos.
9) Elaborar Funções:
Número mínimo de 2 funções de validação de dados, conversão de dados, transformação
de dados.
10) Apresentação:
Apresentação do trabalho em 10 minutos.
